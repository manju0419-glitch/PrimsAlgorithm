Prim's Algorithm

TypeGreedy MST
StructureMin-Priority Queue
Time (heap)O(E log V)
Time (matrix)O(V²)
SpaceO(V + E)
ResultGlobally optimal

Why it's correct

By the Cut Property, the minimum-weight edge crossing any cut between the tree and the rest of the graph is always safe to include in some MST. Prim's repeatedly adds the cheapest edge connecting a new substation to the growing tree — guaranteeing the total cabling cost is the lowest possible while still reaching every point.

Pseudocode

PRIM(G, source):
  key[v] = ∞ for all v; key[source] = 0
  PQ ← all vertices
  while PQ not empty:
    u = EXTRACT-MIN(PQ)
    add edge (parent[u], u) to MST
    for each (u,v,w):
      if v in PQ and w < key[v]:
        key[v]=w; parent[v]=u
